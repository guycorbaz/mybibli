## Business Outcome

**Epic objective NOT fully met.** The application runs but the user cannot complete the core journey (cataloging a book) because:
1. No login page exists — user cannot authenticate
2. i18n system broken in incremental builds — all text shows as raw keys
3. No E2E smoke test validates the real user journey

## Successes

- **Architecture:** Service layer + middleware pattern clean and reusable across all 8 stories
- **i18n from day 1:** EN/FR parallel translations (when build is correct)
- **HtmxResponse + OOB swaps:** Composable pattern for HTMX fragment responses
- **Async metadata pipeline:** BnF provider with skeleton → resolved transition working
- **Soft-delete + optimistic locking:** Infrastructure ready on all entity tables
- **155 unit tests + ~50 E2E:** Comprehensive coverage of implemented features
- **Code review multi-pass:** Caught critical bugs (cookie name mismatch, cache hit logic bug)

## Challenges

- **Login deferred too aggressively** — marked "Epic 7" but blocks all user testing of Epic 1
- **E2E tests inject cookies** — bypass the real authentication flow, masking the login gap
- **i18n proc macro** — `rust_i18n::i18n!` reads YAML at compile time; incremental builds don't re-read changed locale files
- **~60 code review patches** — many issues caught only in review, not by tests
- **Hardcoded "Auteur"** — propagated across stories 1-5, 1-6, 1-7 without correction
- **DRY violation** — `html_escape()` duplicated until story 1-6 extracted it to utils.rs

## Key Insights

1. **Unit tests don't validate user experience** — the complete user journey is the real test
2. **Code review loops catch what tests miss** — but must continue until clean (no Medium+)
3. **Deferred features can be blockers** — login seemed like "nice to have" but is a prerequisite
4. **Build system quirks must be documented** — i18n compile-time embedding is invisible and dangerous

## Team Agreements

1. **An epic is done when the user can accomplish the business objective** — not when stories are checked off
2. **E2E smoke tests must test the real journey** — no cookie injection, no shortcuts
3. **Code review continues until clean** — Medium+ findings trigger another pass
4. **Rebuild after locale changes** — `touch src/lib.rs` before `cargo build`
5. **Extract utilities early** — shared functions go to utils.rs on first use, not after duplication
