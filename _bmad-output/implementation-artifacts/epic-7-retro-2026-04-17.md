## 3. What went well

- **Role matrix as a forcing function.** 7-1's `docs/route-role-matrix.md` converted an ambiguous "which role can do what" into an auditable table the rest of the epic referenced verbatim. `BaseContext` followed naturally — one place to populate, every template composes it. Pattern worth keeping for any future permission surface.
- **CSP scope delivered clean in one shot.** 7-4 touched ~25 files (15 templates, 8 JS, 4 new modules, middleware, audit test). Zero CSP refusals in the enforced-mode E2E pass on first green cycle. The templates-audit `#[test]` becomes a permanent guard — any future inline regression fails `cargo test` before it lands.
- **`scanner-guard.js` + test-only hooks design.** The `window.__MYBIBLI_TEST_HOOKS` sentinel gates `listenerAttachCount`/`listenerDetachCount` / `refreshStack` introspection behind a flag tests flip before loading the script — production bundle carries zero counter overhead, tests get deterministic state. Clean separation.
- **3-layer code review earned its keep.** Every Epic 7 story went through Blind + Edge + Acceptance. 7-5 Edge Case Hunter caught a *real* accessibility regression (Tab / arrow / Ctrl-combos blocked inside modal) that both the dev agent and Test 8 missed. Patched before close.
- **Deep tests caught a spec contradiction.** 7-5 Test 8 (Enter-on-focused-button must not activate) directly contradicted AC 1 bullet 4 as originally written. The refined policy ("text-target inside modal → pass-through; everything else → blocked") was reached by honoring Test 8 and updating AC wording post-implementation (P3 patch). This is exactly what adversarial tests are for.
- **Templates audit pattern compounds.** 7-4 created `src/templates_audit.rs` with 4 regexes. 7-5 added a 5th (`hx-confirm=` allowlist) without touching the walk infrastructure. Next audit will cost ~30 lines, not 200.
- **i18n-aware cookie round-trip (7-3).** Same-origin guard on `?next=` + `SameSite=Lax` cookie + login DB sync — the locale resolution chain reads like architecture doc prose, not spaghetti conditionals.

## 4. Recurring struggles / patterns

1. **AC wording drifted from shipped code in multiple stories.** 7-5 had *three* documented AC deviations (AC 1 text-target policy, AC 3 bare.html retraction, AC 15 line-84 reinterpretation). 7-1 had a policy refinement around `require_role_with_return`. Dev-time judgement was consistently correct and documented in Completion Notes / Change Log, but the AC *text itself* rarely got patched to match reality. 7-5 patched AC 1 via code review P3 — that should become the default move, not a post-review patch.
2. **Per-story commit habit broke again.** Epic 6 retro flagged it. Epic 7 reproduced the exact tangle: 7-4 and 7-5 sat uncommitted in the working tree until Guy asked for "commit et push avant la rétro". One commit, 48 files, +3005/-283, two stories bundled. Retrospective-debt-on-retrospective-debt.
3. **CSRF deferral is now chronic.** Deferred since story 1-2 (2026-03-29). Epic 6 retro action item #3 ("land with auth"). Epic 7 deferred it again across 7-1, 7-3, 7-4 reviews. Four deferrals. The exit criteria — "same-origin form POST + `SameSite=Lax` cookie is the current threat model" — has been re-asserted three times without a formal decision document. Needs a closure: either ship it in Epic 8 or write a one-page threat-model doc that formally accepts the risk.
4. **E2E DB state pollution discovered mid-suite.** 7-5 full-suite run reported 51 failures on the first pass. Root cause: 11-hour-old e2e stack carried DB data from prior runs (V-codes, titles, borrowers accumulated), breaking tests that assume "first scan of this V-code". `docker compose down -v` fixed it, three clean cycles followed. **No test or helper currently guarantees fresh-stack for dev-loop debugging.** CI gets fresh containers per run; local debugging doesn't.
5. **Dev-story + code-review + retro in one long session.** 7-5 went dev → review → patches → done → commit → retro prep in a single agent session. Quality held, but the cadence collapsed the "per-story PR" feedback loop into one big bundle. Epic 6 retro flagged this. Pattern persists.
6. **Policy deviations-to-satisfy-tests.** 7-5 refined its handler policy mid-dev to make Test 8 pass, then patched the AC to match. This is the right direction (test as source of truth), but it means ACs drift from reality between the epic-planning snapshot and the ship. Acceptance Auditor repeatedly flagged these as "partial (defensible)" — good catch mechanism, but the underlying ACs remain stale.
7. **"Known app quirks" in CLAUDE.md lost currency.** The `scanner.ts` STUB line (removed in 7-5) sat stale for multiple epics. The list grows but rarely shrinks. Audit pass after each epic to prune.
