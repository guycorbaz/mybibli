## Challenges

- **TIMESTAMP bug in all dynamic queries** — `sqlx::query()` (dynamic) cannot decode MariaDB TIMESTAMP into Rust NaiveDateTime without explicit CAST. Affected 6 queries across loan.rs and services/loans.rs. The loans page crashed with 500 Internal Server Error whenever a loan existed. Found by Guy during manual testing, NOT by automated tests.
- **DB offline during 3/4 stories** — stories 4-2, 4-3, 4-4 all mention "DB connectivity dropped" due to session.rs query! macro requiring live DB. Caused .sqlx cache files to be deleted, breaking Docker builds.
- **3 Clippy warnings passed through gate** — redundant closures in services/loans.rs. Caused by inability to run full clippy without live DB.
- **6 pre-existing E2E tests fragile** — timing issues with HTMX responses, data isolation between parallel tests, fragile volume edit navigation for non-loanable test
- **AC6 deferred** — "Lend from Title Detail Page" deferred because title page has no volume list yet

## Bugs Found During Retrospective

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| /loans page 500 error with active loans | Dynamic `sqlx::query()` cannot decode MariaDB TIMESTAMP to NaiveDateTime | Added `CAST(loaned_at AS DATETIME)` and `CAST(returned_at AS DATETIME)` to all 6 dynamic queries |
| 3 Clippy warnings | Redundant closures `.map_err(\|e\| AppError::Database(e))` | Changed to `.map_err(AppError::Database)` |
| Docker build failed | .sqlx cache files deleted during DB outage | Regenerated with `cargo sqlx prepare` |

## Key Insights

1. **Dynamic queries are type-unsafe** — `sqlx::query()` (used for dynamic ORDER BY) lacks the type checking of `sqlx::query!` macro. MariaDB type gotchas (TIMESTAMP, BIGINT UNSIGNED, JSON/BLOB) are invisible to dynamic queries. CLAUDE.md gotcha #4 added.
2. **E2E tests gave false confidence** — 17 loan E2E tests existed but never actually passed in Docker because the TIMESTAMP bug prevented the loans page from rendering. The tests were written against a local setup that didn't exhibit the bug.
3. **Manual testing found what automation missed** — Guy's exploratory testing discovered the critical TIMESTAMP bug. As the app grows, manual testing won't scale.
4. **`.sqlx` cache is a process risk** — when DB goes offline during dev, the cache files become stale or deleted. Need `cargo sqlx prepare --check` in CI as a guard.
5. **format! gotcha with # character** — `#scan-result` in format! string is parsed as Rust format specifier. Must use named parameters. (Story 4-3 finding #10)
