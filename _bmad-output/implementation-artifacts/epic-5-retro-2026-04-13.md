## 3. Recurring struggles / patterns (cross-story)

1. **E2E parallel-mode flakes dominated the epic** (5-1, 5-1b, 5-1c = 3 stories, 2 unplanned). Causes spanned: shared `DEV_SESSION_COOKIE` session state, V-code/L-code/borrower collisions, HTMX form-swap races, stale `.feedback-entry.first()` waits, `page.on("dialog")` async registration, and finally a genuine server-side MariaDB deadlock.
2. **`waitForTimeout` anti-pattern accretion.** 5-1c removed 10 occurrences; **32 remain across 9 specs** (`deferred-work.md`). CLAUDE.md bans the pattern but enforcement is review-only.
3. **Mock metadata pollution.** The BnF catch-all returning `"Synthetic TestAuthor"` + `date=2024` for every unknown ISBN caused 5-7's "absent section" case to be unprovable in E2E (moved to unit tests). Same mechanism makes per-spec cleanup fragile.
4. **Hardcoded French labels in SQL / UI.** `'Auteur'` role string reappeared in 5-7's `find_similar` (same bug as 1-5/1-6); hardcoded `Dewey:` literal fixed in 5-8; `Endommagé` select-label in loans spec still deferred.
5. **`manually_edited_fields` is leaky.** 5-8 surfaced that `confirm_metadata` removes the flag whenever the accept checkbox is checked (all 8 `final_<field>` blocks), and `tasks/metadata_fetch.rs::update_title_from_metadata` performs a raw UPDATE with **no version check and no manually-edited guard** — race window where background fetch silently overwrites manual edits.
6. **i18n proc-macro gotcha.** `touch src/lib.rs && cargo build` required after YAML changes — called out in 5-3, 5-6, 5-7, 5-8. Documented in CLAUDE.md, still a trip-wire.
7. **`ORDER BY` without `id ASC` tiebreaker.** 5-8 revealed that `VolumeModel::find_by_location`, `home.rs`, and `active_search` lack a secondary sort key — causes paginated duplicates/skips, especially on Dewey-NULL-heavy sorts.
